# compare — シナリオ比較

同じ契約データに対し、パラメータ（λ, β, 予算など）を変えた複数シナリオを実行し、
優先順位と予算配分がどう変わるかを比較する。

## 手順

1. 比較したい軸をユーザーに確認する（例: λを 0.2 / 0.5 / 1.0 で振る、
   予算を 20M / 40M / 60M で振る、束ねON/OFF）。
2. 各シナリオについて `input/scenario.json` をベースに一時的なシナリオJSONを作り、
   `mid/` に保存する（例 `mid/scenario_lambda02.json`）。
3. それぞれを実行する:
   ```
   python src/Main.py --scenario mid/scenario_lambda02.json
   ```
   （出力が上書きされるため、各実行後に `output/report.md` を
   `mid/report_lambda02.md` などへ退避する）
4. 結果を並べて、ランキング上位と予算配分の差分を表で示す。

## 観点

- λを上げると低確度案件が沈み、手堅い案件が残る。
- βを上げると同一ベンダーの束ねが有利になり、束ねが上位化しうる。
  ただし束ねは確度が下がるため、予算配分では単独高確度案件に劣ることがある。
- 予算を上げると本コミット数が増え、予算不足案件が減る。

比較の狙いは「どのパラメータ設定が正しいか」ではなく、
**判断が各パラメータにどう依存するかを可視化して、IT組織長と握るべき論点を明確にすること**。
