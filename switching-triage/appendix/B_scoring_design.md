# 付録B スコア設計の全因子

スコアは3因子の積：`Score = V × U × R`。以下、各因子の計算式と設計意図。

## B.1 V — リスク調整期待価値

```
net_i   = est_return_i − switch_cost_i
V_raw_i = p_i · net_i · (1 − λ (1 − p_i))
```

- リスクの代理として p_i(1−p_i) を用いる（確度0.5付近が最も曖昧＝高リスク。
  p が0や1に近づくとゼロ）。
- λ を上げるほど、確度の低い案件（1−p_i が大きい）のスコアが割り引かれる
  ＝手堅い案件を好む。これが λ の直感的な効き方。
- V は案件群の中で相対正規化する（最大値で割る、または対数を取る）。
  金額の桁が大きい案件が緊急度・熟度を無視して常に上位に来るのを防ぐ。

### 実行確度 p_i の内訳

```
p_i = contestability_prob · execution_prob
```

両方あれば積を使い、なければ従来の単一確度にフォールバック（後方互換）。
繰り返し延長（延長回数 ≥ 閾値）なら execution_prob を自動で割り引く。

## B.2 U — 緊急度

```
d_i = expiry_months_i − switch_lead_months_i    （着手デッドラインまでの月）
U_i = 0                    if d_i < 0            （着手不能＝失効）
U_i = 1 / (1 + d_i / h)    otherwise             （双曲型）
```

- h はリードタイム基準（月）。
- 双曲型を採るのは、指数型が遠い案件を急激にゼロに潰しすぎるため。緩やかな双曲型なら
  「まだ先だが視野には入れておく」を表現できる。
- d_i < 0 の案件は U=0 でスコアゼロ。「気づいたら満了していた」の取りこぼしを可視化。

## B.3 R — 情報熟度

```
g_i = min( max(d_i, 0) / h,  1 )    （締切が遠いほど不確実性を重く見る）
R_i = 1 − return_sigma_i · g_i
```

- 締切が迫っている（d_i 小）ときは不確実性ペナルティを緩める
  （待つ時間がないなら不完全でも判断する）。
- 締切が遠いときは不確実性を厳しく見て温存に回す。
- これは「待てば不確実性が解ける便益 vs 行使窓の消耗」のトレードオフの、スコア上の表現。

## B.4 4ケースの機械的な振り分け

| ケース | 状況 | 挙動 |
|---|---|---|
| A | 価値高・締切近・情報熟 | 3因子すべて高 → 即着手 |
| B | 価値高・締切遠・情報未熟 | U・R低 → 温存（いずれ浮上） |
| C | 価値中・締切近・情報未熟 | 締切近でR緩和 → 人間が最終判断 |
| D | 価値高・締切超過 | U=0 → 失効を明示 |

同じ数式から機械的に導かれる。これが「いつでも説明できる」の実証。

## B.5 判定ラベル

| ラベル | 条件 |
|---|---|
| 失効 | d_i < 0 |
| 即着手 | score ≥ 0.25 |
| 人間判断 | U ≥ 0.5 かつ R < 0.7 |
| 温存 | Vn > 0.3 かつ U < 0.4 |
| 捨てる候補 | Vn < 0.1 |
| 検討 | 上記以外 |

## B.6 予算配分（第2層の実装）

スコア降順に並べ、手元資金 B の制約下で配分する。

- **本コミット**：確度 p ≥ 0.6 かつ「使用可能額 = B×(1−安全余裕)」に収まる案件。
- **予約枠**：確度 p < 0.6 の案件。B の範囲で仮押さえ。不成立なら次へ回す（リボルビング）。
- **予算不足**：優先度は高い（score ≥ 0.1）が資金が届かない案件。明示する（追加資金の根拠）。
- 失効案件と極小スコア（score < 0.05）は配分対象外。

## B.7 運用上の3注意

1. **因子のスケールを揃える**。U・R は0〜1だが V は金額ベースなので、相対値に
   変換してから掛ける。金額の大きさだけで押し切らせない歯止め。
2. **絶対値でなく順位と分布を見る**。スコアの絶対値に意味を持たせず、降順ランキングと
   上位群・下位群の間の崖（ギャップ）を見る。
3. **σ の入力を必ず取る**。リターンを一つ入れたら確からしさもセットで入れる規律を
   最初に決める。空欄だと R が機能せず温存判断が崩れる。
