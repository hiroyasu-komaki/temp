# 付録A 数理定式化の詳細

## A.1 記号

案件（契約）の集合を i ∈ {1,…,N} とし、各案件は以下を持つ。

- `T_i`：契約満了時期（意思決定のデッドライン）
- `c_i`：切替コスト（準備支度金+移行コスト）の見積り（確率変数）
- `r_i`：切替により得られる想定リターン（確率変数）
- `p_i`：実行確度（俎上に載り、完了に至る確率）
- `σ_i`：r_i, c_i の不確実性（情報の確からしさの逆数）
- `τ_i`：切替に要する期間（リードタイム）
- `B`：手元の運転資金（初回は20百万円）
- `t`：意思決定を行う現時点

**着手可能デッドライン**：案件 i に着手するには、満了 T_i から切替期間 τ_i を逆算した
`D_i = T_i − τ_i` までに意思決定せねばならない。t > D_i の案件は今回は切替不能。

## A.2 第1層：オプション評価

各案件を「行使期限 D_i を持つオプション」とみなす。

```
V_i(t) = max{ p_i · E[r_i − c_i],  Continue_i(t),  0 }
                （今行使する価値）  （待つ価値）    （放棄）
```

「待つ価値」Continue_i(t) がリアルオプションの肝。待てば不確実性 σ_i が解消していくが、
待つほど D_i に近づき行使窓が狭まる。この「不確実性解消の便益」と「行使窓の消耗」の
トレードオフが待機価値を決める。

実務的には、厳密な確率微分方程式まで書かず、残り時間 (D_i − t) と不確実性 σ_i の
2軸で「今判断 / 温存 / 放棄」を振り分ければ十分機能する。厳密なブラック-ショールズ型の
評価はむしろ説明可能性を損なう。

## A.3 第2層：予算制約下の選択（確率版ナップサック）

第1層で「今期着手候補」と判定された集合 S_t に対し、決定変数 x_i ∈ {0,1} で選ぶ。

**チャンス制約**（消費が予算を超える確率を α 以下に抑える）：

```
Pr[ Σ c_i x_i > B ] ≤ α
```

**平均-分散型の目的関数**（期待値からリスクをペナルティとして引く）：

```
max Σ ( p_i E[r_i − c_i] − λ · σ_i² ) x_i
```

λ はリスク回避度。上げるほど「確実な小さい勝ち」を、下げるほど「不確実な大きい勝ち」を
好む。**一件集中か分散かは、λ とチャンス制約から内生的に決まる**。

## A.4 第0層：リボルビングとローリング再最適化

静的な解にしないため、時間の扱いを定義する。

- **トリガー起動の再最適化**：定期的に加えて、(a) いずれかの案件の r_i, c_i, p_i が
  有意に更新されたとき、(b) いずれかの案件が D_i に近づいたとき、(c) 着手済み案件が
  完了して資金が戻ったとき、これらをトリガーに第1層・第2層を回し直す。
- **資金の更新則**：完了・不成立で戻った資金、交渉で使わずに済んだ差額を B に戻し入れ、
  次のトリガー時の制約に反映する。これがリボルビングの数理表現。
- **温存の再評価**：温存案件は σ_i が更新されるたびに待機価値が変わり、D_i が迫れば
  自動的に「今判断」側へ移る。

## A.5 この定式化の限界

1. **r_i の質に全体が依存する**。ゴミを入れればゴミが出る。情報更新こそが生命線。
2. **案件間の独立性を仮定している**。同一ベンダーの複数契約は相互依存があり、
   ベンダー単位のポートフォリオとして括り直す拡張が要る（束ねとして実装済み）。
3. **資源配分の最適化であって、交渉そのものの最適化ではない**。実際のリターンは
   交渉力で決まる部分が大きく、そこはモデルの外側。モデルは「どこに張るべきか」を
   示すが、「張った先でいくら取れるか」は人間の仕事。
